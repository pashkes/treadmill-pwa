# Contributing

Thanks for improving Treadmill Workout PWA. This repository is a small offline-first app, so changes should stay focused and easy to verify.

## Setup

Use npm with Node `>=22.12.0` and npm `>=11.0.0`.

```bash
npm install
npm run dev
```

The dev server binds to `127.0.0.1`.

## Useful Commands

```bash
npm run test          # Vitest unit and component tests
npm run lint          # ESLint with type-aware TypeScript rules
npm run build         # TypeScript build plus Vite production build
npm run test:e2e      # Playwright smoke test
npm run format:check  # Prettier check
npm run format        # Prettier write
```

Run the narrowest useful command while developing. Before marking a code change complete, run `npm run test`, `npm run lint`, and `npm run build`. Run `npm run test:e2e` for changes that affect routing, the live workout flow, Bluetooth behavior, persistence, or PWA behavior.

## Development Guidelines

- Keep completed workouts in Dexie/IndexedDB. Do not add new completed-workout persistence to `localStorage`.
- Keep Zustand for runtime-only state such as active screen, selected workout id, toast state, stats period, locale, Bluetooth status, and live workout state.
- Keep domain modules in `src/domain/` independent from React.
- Use TanStack Router for URL-backed navigation. Do not add TanStack Query.
- Do not hardcode user-facing strings in components. Add translations to `src/i18n/ru.ts`, `src/i18n/uk.ts`, and `src/i18n/en.ts`.
- Prefer native `Intl` APIs for localized dates, labels, and numbers.
- Preserve the mobile-first dark treadmill app style unless a task explicitly asks for redesign.
- Treat Web Bluetooth FTMS control-point support as optional because not all treadmills expose it.
- Do not reintroduce handwritten root `manifest.json` or `sw.js`; PWA behavior is generated by `vite-plugin-pwa`.

## Testing Guidelines

- Add pure logic tests next to domain modules with `*.test.ts`.
- Add component tests next to React components with `*.test.tsx`.
- Dexie tests use `fake-indexeddb` through `src/test/setup.ts`.
- Playwright tests live in `tests/`.
- When fixing a bug, add a regression test when the behavior is practical to automate.

## Pull Request Checklist

Before opening a PR, make sure:

- The change is scoped to the requested behavior.
- Tests were added or updated for meaningful behavior changes.
- `npm run test`, `npm run lint`, and `npm run build` pass.
- `npm run test:e2e` was run when the changed area needs browser smoke coverage.
- i18n files were updated for new or changed user-facing strings.
- Docs were updated if architecture, commands, persistence, routing, or PWA behavior changed.

## Generated And Ignored Files

Do not commit `node_modules`, `dist`, `playwright-report`, `test-results`, or `.worktrees`.
